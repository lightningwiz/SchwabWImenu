SAP Procedures - Negative Inventory

-- SAP Negative Inventory -- Graph
1. Save Dave E. Lewis's daily "NEGATIVE INV - CANNELTON" report here:
    "\\can-dc1\Schwab\S & O P\Negatives\Dave's Daily Negatives report\"
2. Use the daily "NEGATIVE INV - CANNELTON" reports to update the "Negative Inventory Graph"

-- SAP Negative Inventory -- Analize and Repair
1. In SAP first run the "MF47" transaction to find Postprocessing (backflushing) ERRORS to be corrected
2. In SAP run the "COGI" transaction to find Postprocessing (goods movement) ERRORS to be corrected
3. Using a current "NEGATIVE INV - CANNELTON" report or generate a report using "SQ01"
    query name "NEGINV_OLDMAT".
4. Analyze items listed using SAP transactions:
    "MB51" for material number transaction history,
    "MB52" for material number inventory status and quantity by stock room,
    "CS15" for material number where used in BoMs,
    "CS11" for material number BoM, single level and indented,
    "MD04" for material number availability and demand,
    to make appropriate corrections, note actions and comments
5. Make corrections to inventory using the SAP transactions "MB1B", "MFBF", or "CO11N".
    MB1B = Transfer Posting from stockroom to stockroom (311 Movement Type)
    MB1B = Transfer Posting for material substitution (Z09 Movement Type)
    MFBF = Backflushing in Repetitive Mfg. (Production Posting)
    CO11N = Production Order reporting (Descreet Mfg.)
    MB1A = Enter Goods Issue: New Items - REcieve to cost center, to add printed materials(202 Movement Type)
    MB1A = Enter Goods Issue: New Items - GI Scrapping to cost center, to scrap parts or material(Z55 Movement Type)
6. At the same time take the appropriate action to make any changes in SAP to BoMs, routings,
    or the material master that are sources of inventory ERRORS. IE: ECRs, BICs, etc.

- ECR Procedures - Mark -

-- ECR - Engineering Change Request --
1. Suggest Engineering Change Request (Anyone can do this)
2. Forward Engineering Change Request to Mark or Holly
3. Initiate ECR in online ECR system (Mark or Holly)
    Clearly explain what new material numbers need to be added to SAP
4. Wait for approval needed by Product Manager (Bill Buchko)
5. Create Excel Bill of Materials to show changes required (Mark or Holly)
6. After Product Manager approval Product Engineer approves ECR (Mark or Holly)
7. Create any new material numbers in SAP and
    populate only "Basic Data 1" and "Basic Data 2" (Mark or Holly)
8. Create Design Inventory Document (DID), include all new SAP material numbers,
    also include any material number to be obsoleted. (Mark or Holly)
9. Enter SAP material numbers created into ECR system at Admin Screen (Mark or Holly)
    DOUBLE CHECK MATERIAL NUMBERS FOR CORRECTNESS,
    incorret material will cause problems in SAP.
10.Issue Load Sheet Email in ECR system (Mark or Holly)
    Be sure to assign a "ECR Coordinator"
    even if no SAP material numbers are to be uploaded.
11.Complete BOM and DID documents and email to ECR distribution list (Mark or Holly)
12.In "\\chronos\roman\CURRENT ECN's" network folder create a folder with the ECR number and
     brief description (Mark or Holly)
13.Save copy of BOM and DID documents in newly created ECR folder (Mark or Holly)
14.Complete the parts of the load sheets owned (PIC - Primary & Plant, Costing - Primary & Plant).
     For Costing Primary - Accounting 2 - Tax Price 1 check with Accounting (S. Whitmore).
15.When notified that the load sheet is complete and all SAP material numbers have been uploaded,
     check all new material numbers in SAP for accuracy and completeness.
16.Make all BoM and Route changes, double check for accuracy.
17.Perform costing update, mark, release, and notify accounting (Shayne Whitmore).
18. NOTE: "Never reopen a ECR once the ECR has uploaded the material numbers to SAP,
     doing so will generate another upload."
19.When ECR is complete and all transsitions have been made, go into ECR System - Show Activities and check
     completion date, then check CLOSE. Update any relevant logs.
